# Roles

## Server API

Server API should be a replacement for SMTP protocol. It should allow clients to send emails to another server and to receive emails from another server.

## Client API

Client API should be a replacement for IMAP and POP3 protocols. It should allow clients to access their emails and to manage their accounts.

## Database

Everything should be tied together by a database. Could be SQL, NoSQL or anything else, only requirement is that both server and client API must support it.
