# Wanted features

These are the features and qualities that we want our technology to have.

## Security

- MoHTTPS should use strictly HTTPS to prevent MITM attacks and sniffing of data.
- MoHTTPS should be immune to sender spoofing.
- MoHTTPS should be highly resistant to injection attacks.
- MoHTTPS should be highly resistant to any kind of spoofing or forgery.

## Usability

- MoHTTPS should be easy to use, configure, maintain and scale.
- MoHTTPS should have fallback for clients that do not support our technology.
- MoHTTPS should be highly available.
- MoHTTPS should allow running behind any kind of proxy or load balancer.
- MoHTTPS should use as few DNS records as possible.

## Performance

- MoHTTPS should be fast and efficient.
- MoHTTPS should use as few resources as possible.
- MoHTTPS should be able to handle a large number of connections.

## Authenticity, Integrity and Evidentiability

- MoHTTPS should be highly authentic and have a high evidentiary value.
- MoHTTPS should be decentralized.
